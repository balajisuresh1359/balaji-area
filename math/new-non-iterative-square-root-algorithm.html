New Non-Iterative Square Root Algorithm

Posted on: March 9, 2025

I've developed a mathematical formula that calculates square roots with unprecedented accuracy in a single step.

In this method, we need to find the sqaure root of any number using its nearest perfect square.

First, notice that there are \(2n+1\) parts between \(n^2\) and \((n+1)^2\).

This means we can split the interval between \(n\) and \(n+1\) into \(2n+1\) equal parts.

Example: For numbers 4 & 5, \(4^2 = 16\) and \(5^2 = 25\). Between 4 and 5, consider one rectangle with the starting point at 4 and ending at 5. We can split this rectangle into 9 (\(2(4)+1\)) parts equally, each box having a width of \(\frac{1}{2n+1}\). This is a mathematical fact.

Let's consider finding the square root of 18 that lies between 16 and 25. Can we use this logic?

\(4^2 = 16\), and we can have \(2(4)+1 = 9\) parts between 4 and 5. For 18, which is 2 places after 16, would adding two parts give us the square root?

There are 9 parts, and we need two box widths since \(18-16 = 2\), so:

\[4 + \frac{1}{2(4)+1} + \frac{1}{2(4)+1} = 4 + \frac{2}{9} = 4.2222...\]

Squaring this result: \(4.2222...^2 = 17.827160492\), which is not 18.

If a number is between 16 and 25, and we're trying to split the square roots of these numbers (4 & 5) equally, shouldn't the square roots of numbers in this range increase proportionally? Actually, no. This approach is incorrect as we'll see below.

We can't rely solely on these distributions, but we can use them for our calculations.

To find the square root of any number, we need:

  1. The nearest perfect square number that is smaller than our target number. Let's call this \(b\).
  2. We need to create a relationship between these numbers.

Let's call our number \(x\) and its nearest perfect square number \(b\).

Our equation will look like:

\[b + one-number-in-decimal = \sqrt{b^2 + difference-between-b^2-and-x}\]

\[b + a = \sqrt{b^2 + (x - b^2)}\]

where \(a\) is some decimal number we need to find.

If we solve this equation, we get \(b + a = \sqrt{x}\), which doesn't help us proceed further.

So, let's replace \(x - b^2\) with \(m\) (the difference between \(x\) and \(b^2\)):

\[b + a = \sqrt{b^2 + m}\]

Now we can manipulate this equation. First, let's square both sides:

\[b^2 + a^2 + 2ab = b^2 + m\]

Simplifying:

\[a^2 + 2ab = m\] \[a(a + 2b) = m\] \[a = \frac{m}{a + 2b}\]

Here we can substitute the value of \(a\) on the right side infinitely:

\[a = \frac{m}{\frac{m}{\frac{m}{(...) + 2b} + 2b} + 2b}\]

This would go on forever. To break this cycle and simplify our calculations, I'll substitute an approximation for \(a\), which is \(\frac{1}{2b+1}\). This isn't the exact value of \(a\), but it gives a close approximation.

After substitution:

\[a = \frac{m}{\frac{m}{\frac{m}{\frac{1}{2b+1} + 2b} + 2b} + 2b}\]

This can be simplified to:

\[a = \frac{m^2(4b+1) + 4mb^2(2b+1)}{m^2 + 4b(4b^3 + 2b^2 + 3bm + m)}\]

This is our formula to find the value of \(a\) if \(x\) lies between \(b^2\) and \((b+1)^2\).

Let's try an example to make this clear. What is the square root of 52?

The nearest perfect square smaller than 52 is 49, with square root 7.

So, \(b = 7\), \(b^2 = 49\), and \(m = 52 - 49 = 3\).

Substituting these values into our equation:

\[a = \frac{9(28+1) + 4 \cdot 3 \cdot 49 \cdot (14+1)}{9 + 4 \cdot 7(4 \cdot 343 + 2 \cdot 49 + 3 \cdot 7 \cdot 3 + 3)}\] \[a = \frac{9 \cdot 29 + 12 \cdot 49 \cdot 15}{9 + 28(1372 + 98 + 63 + 3)}\] \[a = \frac{261 + 8820}{9 + 28 \cdot 1536}\] \[a = \frac{9081}{43017}\] \[a \approx 0.211102587\]

Now, add \(a\) to \(b\) to find the square root of 52:

\[b + a = \sqrt{x}\] \[7 + 0.211102587 = \sqrt{52}\] \[7.211102587 \approx \sqrt{52}\]

Squaring this result: \(7.211102587^2 \approx 52.00000052\), which is remarkably close to 52!
